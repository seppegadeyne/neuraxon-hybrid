# cuNxon comparison against existing Neuraxon-Hybrid evidence

Verdict: `Target-contract Aigarth fitness remains the strongest strict-contract cuNxon action lane; augmented low-margin training improves hard/standard holdout but still fails stress holdout`

## What changed

Qubic NIA Vol. 8 adds a new hypothesis source for this issue stack: branching ratio / criticality should be treated as an explicit regime diagnostic for Neuraxon and cuNxon. The claim map in `benchmarks/results/qubic_nia_vol8_criticality_claim_map.*` extracts the article's claims, maps them to current artifacts, and recommended a `cunxon-branching-ratio-regime-scan` tracked as https://github.com/sisutuulenisa/neuraxon-hybrid/issues/84. This follow-up now adds `benchmarks/results/cunxon_branching_regime_scan.*`: five fresh Aorus RTX 5090 seeds (`117..121`) all bucketed as `reverberating/near-critical proxy` with mean branching proxy=0.997701, but `stress_holdout` stayed at 0.333333 and 0/5 seeds beat the best constant stress baseline. The boundary remains conservative: branching ratio is a necessary diagnostic, not intelligence evidence unless it correlates with held-out task quality above baselines.

The previous cuNxon slices established CUDA runtime viability, long-horizon raw dynamics, a tiny task-coupled action probe, infer-vs-train sensitivity, hidden-state/pattern inspection, a three-sphere action adapter, a longer task-coupled sweep over 108 live RTX 5090 samples, an interface-semantics probe for absolute output-neuron ports, a supervised motor-target adapter, a source-level C/CUDA audit, an input-proxy target route, an external-drive window probe, a bounded four-hour VRAM-resident dynamics run, a resident task-coupled action probe, an Aigarth readout-semantics probe, a single-seed Aigarth holdout/action probe, a five-seed Aigarth action seed sweep, a hard-holdout/leakage audit, a strict-label audit, a heavy contract-penalty audit, a post-hoc remap audit, a target-contract audit, and target-contract stress/augmented-train audits. This follow-up adds a branching/activity-regime proxy scan rather than another longer smoke run.

The source audit found no desired-output API surface in the public cuNxon execution path. `StepTrain` exposes continuous plasticity and neuromodulator reward injection, but not a label/loss/error-vector argument. External step inputs are scattered only through `sensory_input_ids`, and the membrane kernel treats external input as direct pass-through only for input-class neurons; output neurons are readout-capable but are not directly teacher-forced by putting their absolute ids into `sensory_input_ids`.

The input-port proxy target probe validates the supported side of that boundary. During training, target proxy input ports `[3, 4, 5]` reflected the expected train targets with target proxy train alignment=1.000000. During evaluation, target-proxy drive was removed and motor readout ports `[11, 12, 13]` still stayed `[0, 0, 0]` for all train/holdout cases, decoded to `query`, and scored holdout/overall accuracy `0.333333`, equal to the constant-action baselines on this balanced toy set. This means the external-drive route is observable, but it did not create a useful motor-output adapter.

The external-drive port-window probe sharpens the interface boundary. With a 4/4/4 sphere and identical input vector `[1.0, -1.0, 0.5, -0.5]`, input-class sensory-id windows produced readout/snapshot `(+1, -1, +1, -1)` in both infer and train modes. Hidden and output sensory-id windows stayed `[0, 0, 0, 0]` in both modes, and output-window drive remains non-teacher-forcing. This supports treating public external drive as input-class stimulation only, not as a desired-output/error-channel.

The live upstream 4-sphere reward-only example still runs on Aorus RTX 5090, but the checked run (`./example_4sphere 3000 400 0`) scored 193/400 = `0.4825` overall accuracy against the example's stated `50.0%` chance reference. This is useful source/runtime evidence, not decision-quality evidence.

The longer sweep remains baseline-level. Every mode × horizon accuracy is `0.333333`, exactly equal to the constant-action baselines on the balanced toy set. Shorter frozen infer samples show a little readout diversity, but by `4096`/`32768` steps infer also collapses to `query=9`; both `train` and `train_rewarded` are flat `query=9` at every tested horizon.

The interface probe supports the absolute-neuron-index interpretation used by the C++ path: same-sphere readouts match their full-sphere snapshot slices for both relative input ports `[0..3]` and absolute output ports `[8..11]` in a 4/4/4 sphere. The supervised motor-target follow-up used that route with target ports `[8, 9, 10]`, but the learned/evaluated motor readout still stayed `[0, 0, 0]` for all 6 train/holdout cases.

The earlier snapshot/pattern, multi-sphere, long-sweep and new supervised-target findings now point in the same direction: snapshot APIs expose hidden-state changes and pattern APIs are callable, but recall stayed flat zero; the three-sphere adapter ran but decoded all train/holdout cases to `query` and showed baseline-level holdout accuracy; the explicit supervised motor-target adapter also decoded all cases to `query`; and both action adapters matched trivial constant-action baselines at 0.333333 accuracy.

The VRAM-resident run completed its bounded four-hour window without duplicating the process or hitting an observed GPU resource problem. It produced 16 samples / 4,194,304 infer steps from one resident cuNxon process on the Aorus RTX 5090. The resident readout stayed `[0, 0, 0]` for every sample; active-state occupancy briefly moved from 3/15 to 5/15 and then returned to 3/15 from sample 6 onward; the last eight energy deltas plateaued near `6.7226615e+06`; throughput stayed around 291 steps/sec between samples. This is useful runtime/dynamics evidence, but it still does not measure task quality.

The resident action probe tested the next obvious variant instead of repeating a longer infer-only run. One three-sphere network stayed alive while the runner alternated `StepTrain` on train cases with expected-action neuromodulator pulses, then `StepInfer` evaluation on train and holdout cases without labels for 12 epochs. It produced 72 scored cases, but motor readout stayed `[0, 0, 0]` for every case, decoded to `query`; train, holdout, overall and every epoch accuracy stayed `0.333333`, exactly equal to all three constant-action baselines on the balanced task set. This is task-coupled negative evidence against the simple resident training-loop hypothesis.

The Aigarth readout semantics probe tests a different lower-level hypothesis: perhaps cuNxon supervised signal is expected to enter through the public evolutionary fitness API rather than `StepTrain` labels. The upstream demo-relative readout lane improved margin `1.000000→1.500000`, but those readout ids alias input/hidden neurons in a 4-input/32-hidden/8-output sphere. The absolute-output readout lane also improved margin `0.625000→1.500000`, which is real positive interface/evolution evidence for the public `cunxonNetworkAigarthStep` surface. However, that first Aigarth result was only a two-pattern fitness optimization with no holdout/action score, so it did not overturn the baseline-level Neuraxon-Hybrid action evidence.

The Aigarth holdout/action probe is the first cuNxon action lane in this issue stack to beat the constant-action baselines. It evolved a 3-input/32-hidden/3-output sphere for 18 generations with population size 36, using train cases only in the fitness callback. Final evaluation through `ActionDecoder` scored train, holdout and overall accuracy `0.666667`, while `always_execute`, `always_retry` and `always_query` each scored `0.333333` on the balanced toy set. The result is promising but still narrow: `retry` remained a failure in both train and holdout, decoded actions covered only `execute` and `query`, and this is a tiny engineered train/holdout slice rather than broad generalization or intelligence evidence.

The Aigarth action seed sweep tests whether that first positive signal repeats across fresh cuNxon networks instead of treating one seed as conclusive. Five seed offsets (`82..86`) produced holdout mean=`0.533333`, min=`0.333333`, max=`0.666667`; 3/5 seeds beat the best constant baseline on holdout and 5/5 beat it overall. Aggregate decoded/normalized actions included `execute`, `query` and `retry`, but also unexpected normalized labels: `assertive`. This keeps the Aigarth route alive as a useful follow-up hypothesis while weakening any broad claim: the score is not uniformly repeatable, class behavior is unstable, and the evidence remains a small toy adapter audit.

The cuNxon Aigarth action hard-holdout audit adds a stricter follow-up rather than repeating the same seed sweep. Five seed offsets (`87..91`) again reached perfect train accuracy (`1.000000`) under train-only Aigarth fitness, but harder/noisier holdout mean dropped to `0.500000` with only 2/5 seeds beating the best constant baseline on the hard split. Standard holdout mean was `0.600000`, overall mean was `0.520000`, and the permuted-control leakage/oracle split stayed at `0.000000`, which is a useful sanity check against obvious label leakage but not positive intelligence evidence. Aggregate decoded/normalized actions covered `execute`, `query` and `retry`, but one unexpected `assertive` label remained. This keeps the Aigarth route promising enough for more controlled follow-up, while the `0.500000` train-to-hard-holdout gap and partial seed robustness keep the evidence brittle.

The cuNxon Aigarth strict-label action audit changes the fitness hypothesis instead of merely extending the same sweep. Five fresh seed offsets (`92..96`) used strict-label fitness variant `strict_label_margin`: train-only fitness still rewards correct decoded labels and target alignment, but subtracts an out-of-contract label penalty. The live Aorus RTX 5090 run improved hard-holdout mean to `0.700000` with 4/5 seeds beating the best constant baseline on the hard split, standard holdout mean to `0.733333`, overall mean to `0.626667`, and kept permuted-control mean at `0.000000`. This is a constructive improvement over the prior hard-holdout audit, but not a broad claim: `assertive` still appeared 5 times (`0.066667` unexpected-action rate), and seeds 93/95 stayed weak on holdout/hard-holdout.

The cuNxon Aigarth contract-penalty audit tests whether the remaining out-of-contract `assertive` labels were mainly caused by an underweighted penalty. Five fresh seed offsets (`97..101`) used `strict_label_heavy_penalty`, subtracting three times the unexpected-label rate in the train-only fitness callback. The live run reduced the unexpected-action rate to `0.040000`, but hard-holdout mean dropped to `0.466667`, standard holdout mean dropped to `0.466667`, and only 2/5 seeds beat the best constant baseline on the hard split. Permuted-control mean stayed `0.000000`. This is a useful negative/constraint result: stronger label penalties alone do not improve the Aigarth adapter route and can trade off hard-holdout quality for modest contract cleanup.

The cuNxon Aigarth action remap audit then asks a narrower decoder-contract question without another GPU evolution run: what if the existing live readouts are decoded with the project target-readout contract directly (`readout[0] > 0` -> `execute`, `< 0` -> `retry`, neutral -> `query`) instead of the generic `ActionDecoder` vocabulary. Across 255 replayed train/holdout/hard-holdout/permuted-control cases from the seed sweep, hard-holdout, strict-label and heavy-penalty artifacts, the remap removes `assertive` entirely (`13` original unexpected labels -> `0` remapped unexpected labels). However, aggregate accuracy moves from `0.549020` to `0.537255`, train accuracy drops from `0.916667` to `0.816667`, hard-holdout moves from `0.555556` to `0.544444`, and permuted-control rises from `0.000000` to `0.066667`. So the remap removes assertive labels but does not improve aggregate accuracy; decoder vocabulary was a real contract issue, but not the only bottleneck.

The cuNxon Aigarth target-contract action audit tests the next objective-level hypothesis instead of repeating post-hoc remapping. Five fresh seed offsets (`102..106`) used target-contract fitness (`target_contract_margin`), which decodes train cases with the signed-first-lane project contract and rewards target-readout margin inside the Aigarth fitness callback. Live on Aorus RTX 5090: train mean stayed `1.000000`, holdout mean rose to `0.866667`, hard-holdout mean reached `0.666667`, all three expected labels appeared (`execute=20`, `query=35`, `retry=20`), unexpected labels=0, and permuted-control mean remained `0.000000`. This is the strongest strict-contract Aigarth action result so far, but still a tiny/brittle toy slice: seed `102` stayed at `0.333333` hard-holdout and no broad generalization or intelligence claim follows.

The cuNxon Aigarth target-contract stress audit then asks whether that best lane survives harder low-margin shapes and a second control split. Five fresh seed offsets (`107..111`) again reached train mean `1.000000` and preserved zero unexpected labels, but stress-holdout mean was only `0.333333` with `0/5` seeds beating the best constant baseline on that split. Hard-holdout mean dropped to `0.533333`, standard holdout mean to `0.666667`, and counterfactual-control mean rose to `0.266667` while permuted-control stayed `0.000000`. This weakens any broad generalization claim: the target-contract Aigarth route remains diagnostically useful, but low-margin stress cases are baseline-level.

The cuNxon Aigarth target-contract augmented-train stress audit tests an objective-shaping follow-up rather than increasing penalty weight or repeating a longer run. Five fresh seed offsets (`112..116`) optimized the signed-first-lane target contract on the original train cases plus six low-margin `augmented_train` cases, while `holdout`, `hard_holdout`, `stress_holdout`, `counterfactual_control`, and `permuted_control` labels stayed outside the fitness callback. The live Aorus RTX 5090 result improved standard holdout mean to `1.000000` and hard-holdout mean to `0.866667`, with zero unexpected labels and permuted-control mean `0.000000`. However, stress-holdout mean stayed at `0.333333` with `0/5` seeds beating the best constant baseline, and counterfactual-control mean remained non-zero at `0.066667`. This is constructive objective-shaping evidence for hard/standard holdout, but it confirms that the low-margin stress split remains a bottleneck.

The cuNxon branching-ratio regime scan operationalizes the NIA Vol. 8 criticality hypothesis as a measured diagnostics lane. Five fresh seed offsets (`117..121`) reused the bounded target-contract augmented-train source probe, then derived a coarse branching/activity-ratio proxy from final action-case readout activity. All five seeds landed in a `reverberating/near-critical proxy` bucket with mean branching proxy=0.997701, neutral occupancy between 0.511111 and 0.733333, and readout diversity 5..8. Standard holdout stayed `1.000000`, but `stress_holdout` stayed `0.333333` for every seed and `0/5` seeds beat the best constant stress baseline. This is useful negative evidence for the NIA Vol. 8 map: near-critical-looking proxy dynamics can coexist with baseline-level stress behavior, so the metric is diagnostic rather than sufficient evidence of useful computation.

## Interpretation

cuNxon remains interesting as a GPU runtime and diagnostics surface, and the interface semantics probe removed one ambiguity from adapter design: readout ports should be treated as absolute neuron indices when targeting output neurons. The source audit removes a second ambiguity: output-neuron readout ports are not a public desired-output/teacher-forcing channel. `StepTrain` turns on plasticity, CTSN/DSN updates and neuromodulator-sensitive learning, but source inspection found no API route that supplies desired motor labels directly to output neurons. The Aigarth probes add a constructive lead: the public evolutionary fitness API can improve an absolute-output two-pattern margin and, in the action probes, can beat constant-action baselines on tiny toy action sets. The strict-label audit improved hard-holdout robustness compared with the prior hard-holdout audit and did not introduce obvious permuted-control leakage, but the heavier contract-penalty follow-up did not improve robustness: it reduced the unexpected-label rate while lowering hard-holdout accuracy. The remap audit cleans up the decoder contract post-hoc, and the target-contract audit shows that putting the same signed-first-lane contract into the train-only Aigarth objective can preserve zero unexpected labels while improving holdout/hard-holdout relative to the remap and heavy-penalty lanes. The stress follow-up keeps zero unexpected labels but exposes baseline-level low-margin stress-holdout behavior and a non-zero counterfactual-control score. Augmented low-margin training improves standard holdout and hard-holdout while keeping zero unexpected labels, but the same stress-holdout split remains baseline-level. This remains a brittle toy evidence lane rather than broad generalization.

The supervised motor-target adapter's flat `[0, 0, 0]`/`query` result is therefore best interpreted as an unsupported adapter route, not as proof that all possible cuNxon learning semantics are exhausted. The input-proxy target route confirms target values can be placed on supported input-class ports, but also shows that this alone does not produce non-baseline motor readouts. The external-drive window probe now makes the kernel/source interpretation observable from ctypes: non-input sensory-id windows do not act like target channels in this controlled setup. The completed VRAM-resident run makes "just keep the same small process alive longer" a weak standalone hypothesis for this exact 3/9/3 infer-only setup: it produced stable resource/runtime evidence and some early occupancy movement, but no readout diversity. The resident action probe adds explicit task scoring to the resident-process idea and is also negative: repeated resident train/eval epochs with reward pulses still collapse to the same `query` motor readout and do not beat baselines. The next useful direction is to stress the Aigarth action route with more classes/seeds/holdout shapes and guard against fitness leakage/oracle behavior, not another longer run of the same resident or `StepTrain` loops.

## Evidence boundary

This comparison deliberately separates runtime viability and interface semantics from decision quality. Snapshot activity, callable pattern APIs, inter-sphere topology construction, longer horizons, reward injection, absolute-neuron-index port mapping, explicit but unsupported output-port target attempts, supported input-class target proxy drive, source-level absence of a desired-output API surface, the completed VRAM-resident process/state artifacts, and the Aigarth two-pattern margin improvement prove useful diagnostic boundaries. The Aigarth action probes are stronger than the prior flat action lanes because they beat constant-action baselines in tiny engineered action setups and partially repeat across seeds; the strict-label variant improved the hard-holdout mean to `0.700000`, while the heavy contract-penalty variant dropped to `0.466667` despite reducing unexpected labels to `0.040000`. The signed-first-lane remap removes all unexpected labels in replay, but it lowers aggregate replay accuracy to `0.537255`, so it is decoder-contract evidence rather than a decision-quality improvement. The target-contract Aigarth objective remains the current best tiny action lane, but the stress audit is important negative evidence: stress-holdout mean is only `0.333333`, `0/5` seeds beat the best constant baseline on stress_holdout, and counterfactual-control mean is `0.266667`. The augmented-train target-contract follow-up improves holdout to `1.000000` and hard-holdout to `0.866667` with zero unexpected labels, but it still leaves stress-holdout at `0.333333` and counterfactual-control at `0.066667`; objective shaping helped some splits without solving the low-margin stress bottleneck. These target-contract results do not prove intelligence: the task is tiny, seed behavior remains uneven, and harder/noisy holdout/control audits expose brittleness. The flat recall, below-chance upstream reward-only demo accuracy in this run, baseline-level Hybrid action accuracy outside the Aigarth fitness route, resident-process dynamics, resident task loop, and brittle Aigarth action coverage remain important caveats.
